# 1:1 문의 상태/미읽음 기능 테스트 시나리오

기준: `inquiry_list.php` (좌측 목록 + 우측 채팅) 화면

## 1) 사용자 문의 생성 → NEW 노출
1. 일반 사용자 계정으로 로그인
2. 1:1 문의 > `문의 보내기`로 문의 생성
3. 1:1 문의 목록 상단 `새 문의` 탭에 생성한 문의가 표시되고, 목록 항목에 `NEW` 배지가 보인다.

## 2) 관리자 첫 답변 → IN_PROGRESS 자동 전환
1. 관리자 계정으로 로그인
2. 1:1 문의 > `새 문의` 탭에서 해당 문의를 열고 메시지를 1건 전송
3. 해당 문의의 상태가 `진행중(IN_PROGRESS)`로 자동 전환된다.

## 3) 사용자 새 메시지 → 관리자 목록에 미읽음 표시
1. 사용자 계정으로 로그인
2. 방금 진행중으로 전환된 문의에 메시지를 추가로 전송
3. 관리자 계정으로 다시 로그인
4. `진행중` 탭에서 해당 문의 목록 항목에 `미읽음 N` 배지가 표시된다.

## 4) 관리자 해결 처리(사유 입력) → RESOLVED + 시스템 메시지
1. 관리자 계정으로 해당 문의 우측 상단 `해결 처리` 클릭
2. 사유를 선택(필수), 메모는 선택 입력 후 `해결 처리` 확정
3. 상태가 `문의 해결(RESOLVED)`로 바뀌고, 채팅에 시스템 메시지가 1건 자동으로 추가된다.
   - 예: `관리자에 의해 문의가 해결 처리되었습니다. (사유: 사용자 응답 없음)`

## 5) 사용자 재오픈 → IN_PROGRESS
1. 사용자 계정으로 `문의 해결` 탭에서 해당 문의를 열기
2. `다시 열기` 클릭
3. 상태가 `진행중(IN_PROGRESS)`로 전환된다.

## 참고(재오픈 기간)
- 정책: 해결 후 7일 이내만 재오픈 가능
- 해결 시각(`resolved_at`) 기준으로 7일이 지난 문의는 재오픈 시 오류 메시지가 표시된다.
